# Audience matrix (rule A)

Set these four coordinates before the governing thought. Each one forces
concrete adjustments to depth, ordering, and density. Pick a value per axis,
then apply the adjustments. State the coordinates and the adjustments out loud
so the user can correct them.

The axes are independent - a deck is one value on each. The combination is what
shapes the deck, not any single axis.

---

## 1. Altitude - who is in the room

| Value | What it means | Deck adjustment |
|---|---|---|
| **Executive** | Decision-makers, time-poor, want the answer | Answer first and hard (L1). Fewer slides. Method, assumptions, and working go to an appendix. Every slide survivable in 10 seconds. |
| **Technical** | Peers who will interrogate the method | Method and assumptions stay in the body, not the appendix. Show the working. Defend the numbers in place. |
| **Mixed** | Both in the room | Lead executive, layer technical. Main line answer-first; technical depth in clearly-marked backup the experts can pull you into. |

## 2. Purpose - what the deck is for

| Value | What it means | Deck adjustment |
|---|---|---|
| **Decide** | The audience must make a call | Frame the decision explicitly. Lay out the options, the recommendation, the trade-offs, and the ask. The deck is incomplete without a clear "what we need from you". |
| **Persuade** | Move them to a position | Build tension: what is vs what could be. Front-load the cost of the status quo. Evidence escalates toward the conclusion. |
| **Inform** | Update, no decision sought | Drop the ask. Organise for reference and recall, not pressure. Density can be higher; the reader is browsing, not being moved. |

## 3. Disposition - how they arrive

| Value | What it means | Deck adjustment |
|---|---|---|
| **Friendly** | Predisposed to agree | Lighter evidence load. Don't over-justify; get to the point and the ask. |
| **Neutral** | No prior position | Balanced evidence. Make the case cleanly; acknowledge the obvious alternative. |
| **Skeptical** | Predisposed to push back | Front-load evidence and assumptions. Pre-empt the objections - name them before they do. Show the data behind every claim. Earn each step before taking the next. |

## 4. Delivery - how it reaches them

| Value | What it means | Deck adjustment |
|---|---|---|
| **Presented live** | You are there to narrate | Slides can lean on you. Less on-slide text; the title plus exhibit is enough because you fill the gaps. |
| **Sent standalone** | Read with no presenter | The deck must be fully self-contained. Titles carry the entire story (L2 is non-negotiable). Every exhibit self-explaining. Density rises because nothing is narrated. This is the hardest mode - assume the reader skims titles only. |

---

## Reading the combination

The coordinates compound. A worked example for a common consulting shape:

**Executive / Decide / Skeptical / Sent** - a senior investment or business case
read cold by people inclined to say no:

- Answer first, stated as a recommendation with the ask (Executive + Decide).
- Options and trade-offs framed explicitly (Decide).
- Evidence and assumptions front-loaded, objections pre-empted (Skeptical).
- Fully self-contained, titles tell the whole story, no reliance on a presenter
  (Sent).

That specific combination produces a dense, answer-first, evidence-heavy,
self-contained deck. Change one coordinate and the shape changes: the same
content **Presented live** sheds on-slide text and leans on narration; the same
content for a **Friendly** audience drops the heavy pre-emption.

When the user can't name a coordinate, infer it from the context they gave,
state the assumption, and move on - don't stall the interview.
